# 문제 해결

> [English](TROUBLESHOOTING.md)

## 먼저 확인하세요

1. 테스트하려는 Codex Desktop, Stream Deck, ThreadDeck을 최신 버전으로 맞춥니다.
2. **시스템 설정 → 개인정보 보호 및 보안 → 손쉬운 사용**에서 **Stream Deck**을 허용합니다.
3. 손쉬운 사용 권한을 바꿨다면 Stream Deck을 완전히 종료하고 다시 실행합니다.
4. ThreadDeck 설치 뒤 Codex를 한 번 종료·실행해 Micro 렌더러 연결을 붙입니다.
5. 폴백 경로를 위해 Codex에 마이크 권한이 있고 **받아쓰기 시작**이 `Control+Shift+D`인지 확인합니다.
6. 음성 버튼을 시험할 때는 Codex를 열고 작성창이 보이게 둡니다.

ThreadDeck에는 화면 기록이나 전체 디스크 접근 권한이 필요하지 않습니다.

## ThreadDeck 프로필이 보이지 않습니다

Neo 프로필은 플러그인과 함께 설치되지만 현재 프로필을 강제로 바꾸지는 않습니다. Stream Deck 앱 상단의 프로필 선택기에서 **ThreadDeck for Codex**를 선택하세요. 목록에 없다면 최신 `.streamDeckPlugin` 패키지를 다시 설치하고 Stream Deck을 재시작합니다.

## Effort/Fast에 `Codex 재실행` 또는 `Micro 점검`이 표시됩니다

ThreadDeck은 Codex를 자동으로 종료하거나 재실행하지 않습니다. 현재 프로세스에 건강한 루프백 브리지가 없으면 그대로 두고 **Codex 재실행**을 표시하므로, 편한 때에 사용자가 직접 Codex를 종료하고 다시 여세요. 디버거를 LAN에 바인딩하지 않습니다.

소스 체크아웃에서 `pnpm run doctor`를 실행하면 읽기 전용 상태를 볼 수 있습니다. `연결됨`은 렌더러 endpoint와 메인 `app://` 대상이 모두 응답했다는 뜻이고, `Codex 재실행`은 현재 프로세스에 건강한 브리지가 없다는 뜻이며, `중지`는 Codex가 닫혀 있다는 뜻입니다. doctor와 ThreadDeck 모두 Codex를 시작하거나 종료하지 않습니다.

연결이 없는 동안에도 검증된 손쉬운 사용·단축키 어댑터는 유지됩니다. Codex 업데이트 뒤 내부 동작 하나만 실패하면 반복해서 누르지 마세요. 렌더러에 전달됐는지 모호한 동작은 Fast 두 번 토글이나 중복 제출을 막기 위해 폴백으로 재실행하지 않습니다. ThreadDeck을 업데이트하거나 Codex 버전과 사용한 버튼을 알려주세요.

## 단축키·미디어·원격 전환이 모두 동작하지 않습니다

ThreadDeck은 시작할 때와 30초마다 권한과 기존 제어 단계를 따로 확인합니다.

- **`권한 필요`** — Stream Deck의 macOS 손쉬운 사용 권한이 없습니다.
- **`입력 권한`** — 손쉬운 사용 스위치가 켜져 보여도 macOS가 합성 키 입력을 버리고 있습니다.
- **`Codex 권한`** — 두 사전 검사는 통과했지만 읽기 전용 Codex AX 확인이 두 번 연속 실패했습니다.
- **`Codex 점검` / `입력 점검`** — 권한 검사는 통과했지만 해당 실제 동작이 30초 안에 두 번 실패했습니다.
- **`미디어 점검`** — 활성 미디어 앱은 찾았지만 재생 상태 또는 일시정지 명령을 확인하지 못했습니다.

권한이 없으면 ThreadDeck이 10분에 한 번 이하로 macOS 공식 권한 요청을 실행하고 버튼 경고를 유지합니다. 복구 여부를 자동으로 다시 검사하며 정상화되면 Stream Deck 성공 표시와 함께 경고를 해제합니다. 스위치가 이미 켜져 있는데도 `Codex 권한`이 유지되면 손쉬운 사용 목록에서 Stream Deck을 제거했다가 다시 추가하고 Stream Deck을 완전히 종료·재실행하세요. `점검` 경고라면 권한보다 Codex UI나 단축키 호환성 문제이므로 사용 중인 버전과 함께 제보하는 편이 정확합니다.

## 작업 버튼을 길게 눌러도 녹음이 시작되지 않습니다

작업 버튼에는 두 경로가 있습니다.

- **0.55초 전**에 놓으면 작업만 엽니다.
- **0.55초를 지나** 계속 누릅니다. 원격 전환이 느리면 먼저 `전환 준비`가 표시될 수 있으므로 카드에 `말하는 중`이 나타날 때까지 누르고, 누른 채 말한 뒤 놓으면 받아쓰기와 자동 제출을 진행합니다.

오류가 표시되면 Codex를 열 수 있는지, 작성창이 보이는지, 마이크 권한이 있는지, 받아쓰기 시작이 `Control+Shift+D`인지 확인하세요. Codex 오디오 입력이 실제로 시작되지 않으면 ThreadDeck은 잘못된 녹음 상태를 유지하지 않고 오류로 전환합니다.

입력 소스를 영어로 바꿀 필요는 없습니다. ThreadDeck은 한글 입력 상태에서도 물리 D와 라틴 `D`를 함께 전송합니다.

## 전용 마이크가 글만 남기고 전송하지 않습니다

정상 동작입니다. 전용 마이크는 검토 우선 누르는 동안 말하기 버튼입니다. 누르는 즉시 녹음을 시작하고, 말하는 동안 계속 누른 뒤 놓으면 Codex 작성창에 초안을 남깁니다. 자동 전송이 필요하면 작업 버튼을 길게 누르거나 이후 보내기 버튼을 사용하세요.

ThreadDeck은 미디어 앱 허용 목록을 사용하지 않습니다. 실제 Core Audio 출력 프로세스의 화면 앱을 자동으로 찾고, 의미가 확인된 재생·일시정지 컨트롤만 누르며, 확인할 수 없을 때만 macOS 정상 미디어 명령으로 보수적으로 폴백합니다. 멈춘 앱의 번들 식별자는 10분짜리 로컬 임대 정보에만 보관하므로 마지막 음성 버튼을 놓을 때 자신이 멈춘 앱만 재개합니다. Apple Music, Chrome YouTube, Safari YouTube를 실기 검증했으며 프로세스를 강제로 멈추거나 PID·제목·URL·미디어 문구를 저장하지 않습니다. 브라우저 세션은 여러 탭을 대표할 수 있습니다. `미디어 점검`이 계속 보이면 재생 중인 탭이나 앱을 한 번 앞으로 가져오고 Stream Deck 손쉬운 사용 권한을 확인하세요.

## 보내기 버튼이 다른 단축키를 보냅니다

두 동작 모두 버튼을 놓을 때 실행됩니다.

- **0.6초 전**에 놓기 → Return;
- 버튼이 **파란색**으로 바뀔 때까지 누른 뒤 놓기 → Command+Return.

파란색으로 바뀌지 않으면 보내기 액션을 제거했다 다시 추가하거나 최신 플러그인을 다시 설치하세요.

## 미디어 페이지에서 앱이 종료됩니다

기본 앱 실행 버튼 4개는 Elgato의 Open Application 액션이며 길게 누르기 동작이 **종료**로 설정돼 있습니다. Stream Deck, Music, Chrome, Codex를 열거나 앞으로 가져오려면 짧게 누르세요. 버튼을 계속 누르면 Stream Deck이 설정된 앱 종료 동작을 실행합니다. Stream Deck 앱에서 대상 앱이나 길게 누르기 설정을 바꿀 수 있습니다.

## 작업 카드가 비어 있습니다

Codex Desktop을 열고 작업을 하나 이상 만듭니다. 사용자 로컬 고정·최근 작업이 목록을 채워야 합니다. Codex 업데이트가 로컬 DB 형식을 바꿨다면 실제 작업 제목을 뺀 버전 정보와 개인정보를 제거한 Stream Deck 플러그인 로그만 이슈에 적어주세요.

## 원격 작업이 목록에 없습니다

ThreadDeck은 고정하지 않은 원격 기록을 의도적으로 제외합니다. 원격 작업 하나를 표시하려면 다음 순서로 진행하세요.

1. Codex에서 해당 컴퓨터를 한 번 열어 Desktop이 요약을 캐시하게 합니다.
2. 표시할 작업을 Codex에서 직접 고정합니다.
3. 다음 ThreadDeck 갱신을 기다립니다.

고정을 해제하면 하드웨어 목록에서도 빠집니다. 내부 보조·검토 작업은 항상 제외됩니다.

## 원격 작업이 열리지 않습니다

원격 선택에는 Stream Deck 손쉬운 사용 권한이 필요합니다. ThreadDeck은 화면 좌표를 클릭하지 않고 정확한 Codex 사이드바 또는 통합 검색 결과를 키보드로 활성화합니다.

같은 작업 버튼을 반복해서 누르면 진행 중인 전환 하나를 공유합니다. Codex가 노출한 정확한 작업 UUID를 우선하고 손쉬운 사용 사이드바 컨트롤을 먼저 시도하며, 사용할 수 없을 때만 통합 검색을 한 번 엽니다. 가벼운 포커스 헤더 검사가 검증된 결과를 적응형으로 기다립니다. 그사이 다른 작업을 누르면 오래된 전환은 취소되고, 이미 알려진 같은 제목은 제목만으로 열지 않습니다.

- **손쉬운 사용** — 권한이 없습니다.
- **제목 중복** — 같은 제목의 원격 결과가 여러 개입니다. 이름을 바꾸거나 하나의 고정을 해제하세요.
- **원격 확인** — Codex가 캐시된 컴퓨터나 작업을 더 이상 노출하지 않습니다. Codex에서 해당 컴퓨터를 다시 여세요.

## 원격 소요 시간이 알 수 없음으로 표시됩니다

ThreadDeck은 캐시된 요약 시각을 완료 이벤트로 사용하지 않습니다. 신뢰할 종료 이벤트나 실시간 종료 전환을 관찰하지 못했다면 임의의 짧은 시간을 만들지 않고 알 수 없음으로 표시합니다. 작업 진행 중 해당 작업을 열어 두면 실시간 시간 감지가 가장 정확합니다.

## `상태를 읽지 못함`이 깜빡이거나 목록을 덮습니다

최신 릴리스로 업데이트하세요. ThreadDeck은 짧은 읽기 실패를 재시도하고 마지막 정상 목록을 유지합니다. 처음부터 정상 목록을 한 번도 읽지 못했을 때만 3회 연속 실패 후 오류를 표시합니다. 계속되면 Codex와 Stream Deck을 재시작하고 실제 작업 제목 없이 버전만 보고하세요.

## 완료된 작업 버튼만 반짝입니다

최신 빌드로 업데이트하세요. 첫 완료 프레임은 후속 프레임을 묶기 전에 화면의 모든 ThreadDeck 소유 버튼에 전송됩니다. 미디어 페이지의 앱 실행 버튼처럼 Elgato 소유 액션은 원래 렌더링을 유지하므로 오버레이를 받을 수 없습니다.

## 대기열 배지가 나타나지 않습니다

Codex에서 해당 작업을 열어 대기 동작이 보이게 하고 손쉬운 사용 권한을 확인하세요. ThreadDeck은 현재 열린 작업을 관찰한 뒤 개인정보 안전 개수만 해당 카드에 유지합니다. 한국어·영어 Codex 손쉬운 사용 라벨을 인식하며 대기 메시지 본문은 읽지 않습니다.

## 닫은 사이드챗이 다시 나타납니다

최신 릴리스로 업데이트하세요. 임시 사이드챗은 현재 Codex App Server 세션으로 제한되고 상태 파일이 잠깐 다시 쓰여도 닫힘 기록을 유지합니다. 보관된 ID를 포함한 영구 작업은 프롬프트 기록에서 사이드챗으로 다시 들어오지 못하게 차단합니다.

## 한도 버튼에 숫자가 없습니다

CodexBar를 설치하고 터미널에서 아래 명령이 작동하는지 확인합니다.

```sh
codexbar usage --format json
```

ThreadDeck은 `CODEXBAR_PATH`, `~/.local/bin/codexbar`, `/opt/homebrew/bin/codexbar`, `/usr/local/bin/codexbar`, `PATH` 순서로 찾습니다. 모든 ThreadDeck 페이지에서 한도를 미리 읽고 일시적인 실패 중에는 마지막 정상 값을 유지합니다.

## 라이트·다크 모드가 잘못 표시됩니다

ThreadDeck은 2초마다 시스템 화면 모드를 따릅니다. 디버깅할 때는 Stream Deck을 `THREADDECK_APPEARANCE=dark` 또는 `THREADDECK_APPEARANCE=light` 환경 변수와 함께 실행할 수 있습니다.

## Codex 데이터 위치를 바꿨습니다

`CODEX_HOME`을 설정하거나 `THREADDECK_STATE_DB`, `THREADDECK_GLOBAL_STATE`, `THREADDECK_SESSION_INDEX`, `THREADDECK_PROCESS_REGISTRY`로 각 경로를 지정합니다.

## 개인정보를 지키며 문제를 보고하는 방법

macOS, Codex Desktop, Stream Deck, ThreadDeck, 장치 모델 버전과 사용한 버튼·제스처를 적어주세요. 개인정보 안전 ThreadDeck 기록은 `~/Library/Logs/ThreadDeck/runtime.jsonl`에 있으며 작업 제목·ID를 포함하지 않고 256 KiB로 제한됩니다. Codex Desktop 원본 로그, 세션·DB 파일, 실제 작업 제목, 원격 호스트 이름, 장치 식별자는 첨부하지 마세요. [보안과 개인정보 안내](SECURITY.ko.md)를 참고하세요.
