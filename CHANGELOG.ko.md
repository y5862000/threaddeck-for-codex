# 변경 기록

> [English](CHANGELOG.md)

## 미배포

- 확인하지 않은 완료 상태를 추가했습니다. 완료된 작업 버튼은 플러그인을 다시 시작해도 느린 초록 펄스를 유지하며, 전환 성공 또는 Codex 전면의 정확한 작업 일치가 확인되어야 해제됩니다. 전환에 실패하면 알림을 그대로 남깁니다.
- 대시보드의 현재/마지막 전환 작업 버튼을 **현재 작업**으로 바꿨습니다. 이제 Codex 앱에서 직접 선택한 작업도 내부 활성 창을 기준으로 따라가며, 받아쓰기·제출 안전성을 위한 전면 작업 검증은 더 엄격한 기존 경로를 그대로 사용합니다.

## 0.3.2-beta — 2026-07-19

- Fast mode 버튼의 상태 읽기·반전·적용을 하나의 네이티브 트랜잭션으로 합쳐, 한 번 누를 때 모델 메뉴가 여러 차례 나타나던 버벅임을 없앴습니다.
- 음성 입력 시작 시 CoreAudio 프로세스가 살아 있다는 이유만으로 재생/일시정지 키를 보내지 않습니다. macOS의 현재 미디어가 실제 재생 중이라고 확인된 경우에만 멈추므로, 사용자가 이미 일시정지한 음악이나 영상이 녹음 시작과 함께 재생되는 문제를 막았습니다. 상태를 확실히 읽지 못할 때도 미디어를 건드리지 않습니다.
- Fast mode 버튼의 정상 켬·끔 상태에서는 문구를 없애고 아이콘만 표시하며, 확인이 필요한 상태에만 경고 문구를 남겼습니다. 번개의 꼭대기도 뾰족한 실루엣으로 다시 그려 위가 잘린 듯 보이지 않게 했습니다.
- 작업 전환이나 버튼 표시를 위한 Fast mode 조회가 모델 선택기를 열지 않도록 완전히 읽기 전용으로 분리했습니다. Codex가 닫힌 선택기의 번개 SVG를 손쉬운 사용 값에 포함하지 않을 때는 정확한 작업의 `service_tier` 기록과 마지막 검증값을 유지하며, 실제 메뉴는 사용자가 Fast mode 버튼을 눌렀을 때만 엽니다.

## 0.3.1-beta — 2026-07-19

- 배포 패키지 설치 과정에서 네이티브 `keybridge`의 실행 권한이 사라지는 문제를 수정했습니다. 이제 Node 진입점이 어떤 플러그인 모드를 시작하기 전에 실행 권한을 복구하고 확인하며, 릴리스 검증에서 실제 설치본과 같은 `0666` 상태를 재현해 자동 복구가 실패하면 패키징도 실패합니다.
- 최신 Codex 모델 메뉴에 맞게 Fast mode 경로를 수정했습니다. Chromium이 `AXPress` 성공을 반환하면서 메뉴를 열지 않는 경우 정확한 손쉬운 사용 컨트롤에 키보드 포커스를 준 뒤 Return으로 열고, `표준 기본 속도`·`빠름 1.5배 속도` 같은 한글 설명형 옵션을 인식해 마우스 좌표 없이 선택하며, Escape로 메뉴를 닫고 바뀐 실제 상태까지 확인합니다.
- 고정된 pnpm 10 도구와 최신 pnpm 클라이언트 양쪽에서 `sharp` 빌드만 명시적으로 허용하도록 프로젝트 설정을 보강했습니다.

## 0.3.0-beta — 2026-07-19

- 기존 현재/마지막 전환 작업 액션을 다른 의미로 바꾸지 않고, 별도로 선택할 수 있는 **상위 작업 1** 액션(`com.yechan.threaddeck.thread.top1`)을 추가했습니다. 기본 작업 페이지는 상위 작업 1로 시작하고 대시보드는 현재/마지막 전환 작업과 Fast mode를 유지하며, 두 작업 액션 모두 짧게 눌러 열기와 길게 눌러 음성 입력하기를 지원합니다.
- 원격 작업의 추론 강도와 서비스 속도 결합을 복구했습니다. 정확한 작업과 턴에 묶인 작성창 관찰값을 갱신 뒤에도 유지하되 의미가 모호한 요약의 `mode` 필드를 속도로 추정하지 않으며, 예전의 항상 보이는 컨트롤이 없다는 이유만으로 고속·일반 모드 액션 아이콘이 사용 불가로 떨어지지 않습니다.
- 대기열 관찰과 완료 판정을 분리했습니다. 대기 명령을 수정·삭제하거나 실행 순서로 넘겨도 작업 버튼과 전체 버튼의 초록 펄스가 시작되지 않습니다. 대기열이 안정적으로 비어 있고 마지막 턴의 새 종료값이 확인될 때만 펄스가 나타나며, 원격의 늦은 이전 턴 종료값은 대기열 전환 시점에서 차단하되 실제 후속 턴이 끝나면 정상적으로 완료를 알립니다.
- 끝나지 않은 목표가 있는 작업 카드에 Lucide Goal 과녁을 추가하고, 시간을 마지막 턴이 아니라 Codex의 목표 전체 누적 시간으로 바꿨습니다. 로컬 목표는 읽기 전용 목표 DB에서, 화면에 열린 원격 목표는 개인정보 안전한 고정 문구 손쉬운 사용 감지와 목표 내용이 없는 7일 캐시로 읽습니다. `active`에서만 시간이 증가하고 차단·일시중지·한도 제한·달성·중단·오류·대기에서는 최초 값에 고정하며, 토큰 예산형 목표도 배지를 유지하고 새 자동 후속 턴에서는 임시 동결을 풀 수 있게 했습니다.
- 대시보드 `1,1`의 기본 앱 전환 버튼을 Codex Fast mode 전용 토글로 교체하되 앱 전환 액션은 사용자 지정 레이아웃용으로 유지했습니다. 대시보드의 현재/마지막 전환 작업 버튼은 현재 작업 또는 ThreadDeck이 전환 성공을 확인한 마지막 작업을 명시적으로 나타냅니다.
- 한 번의 물리 버튼 입력 안에서 로컬 작업과 사이드챗 딥 링크를 재시도하고 실제 대상 전환을 확인한 뒤에만 성공으로 처리합니다. Fast mode는 버튼이 나타날 때, 확인된 작업 전환 뒤, 누를 때마다 작성창의 실제 상태를 다시 읽고 늦은 네이티브 확인을 복구하며, 작성창을 바꾸는 동작과 직렬화해 다른 작업으로 잘못 적용되지 않게 했습니다.
- 받아쓰기 종료를 실패 폐쇄형 자동 복구 경로로 바꿨습니다. Codex 오디오 입력이 꺼졌다고 확인되기 전에는 새 녹음·자동 제출·미디어 재생을 시작하지 않으며, 늦게 나타나는 중지 컨트롤과 일시적인 CoreAudio 지연은 전역 단일 세대 게이트에서 안전하게 재시도합니다. 중지 이후 시작으로 바뀐 컨트롤은 다시 누르지 않고, 시작 확인값이 계속 불명확해도 정상 홀드를 취소하지 않으며, 빠른 연속 입력의 미디어 재개를 디바운스해 짧은 재생 끊김을 줄이고 종료 정리도 한 번만 실행합니다.
- 같은 원격 작업 버튼을 반복해 눌러도 하나의 전환만 공유하고, 다른 작업을 새로 누르면 오래된 시도를 취소하도록 변경했습니다. 정확한 작업 UUID를 정규화된 제목 지문보다 우선하고, 이미 알려진 같은 제목은 엄격한 UUID 일치를 요구하며, 통합 검색은 가벼운 적응형 준비 확인과 함께 최대 한 번만 열고 입력합니다.
- 원격 작업과 작성창이 준비되는 동안 작업 버튼의 길게 누르기 의도를 유지하고 `전환 준비` 상태를 표시합니다. 정확한 대상 작성창을 확인하지 못하면 자동 제출하지 않으며, 이미 놓은 느린 입력이 다음 버튼 입력 상태를 지우지 못하도록 막았습니다.
- 음성 입력 중 프로세스를 강제로 멈추던 방식을 macOS의 정상 재생·일시정지 명령으로 교체했습니다. 여러 음성 버튼이 겹쳐도 비동기 소유권을 균형 있게 관리해 더 자연스럽게 멈추고 마지막 녹음 버튼을 놓은 뒤에만 재생합니다.
- 빠른 서비스의 번개 표시를 진행 중·중단·대기·오류 카드에 유지하되 완료 카드는 체크 표시에 집중하도록 번개를 숨기고, 번개·원격 검색 대체 경로·길게 누르기 경합·대상 확인·미디어 소유권 회귀 검증을 추가했습니다.
- 영문·한글 사용자 안내를 설치와 물리 버튼 제스처 중심으로 전면 개편하고, 종합 애니메이션이 실제 작업 버튼 자동 제출 경로를 표시하도록 바로잡았습니다. 작업 버튼 음성 입력, 전용 마이크 초안 받아쓰기, 파란색 보내기 길게 누르기, 기본 앱 실행 버튼의 길게 눌러 종료 동작 GIF를 재현 가능하게 추가했으며 Stream Deck 액션 툴팁에도 같은 임계 시간과 놓기 동작을 명시했습니다.
- `thread_source`, `source.subagent`, `agent_path` 구조 메타데이터로 Codex 서브에이전트·guardian 검토 행을 모든 작업 버튼 데이터 소스에서 제외하고, 구버전에는 정확한 내부 주입 제목 서명을 보조 필터로 적용했습니다. 숨긴 영구 작업 ID가 임시 사이드챗으로 다시 들어오는 경로도 차단했습니다.
- 원격 작업의 reasoning-summary를 턴·시각으로 검증한 뒤 원문 없이 `계획 중`, `분석 중`, `구현 중`, `검증 중`, `실행 중`, `정리 중` 같은 중간 단계로 표시합니다. 대기·오류·완료 상태는 계속 우선하며 오래되거나 다른 턴의 단계는 표시하지 않습니다.
- 로컬은 고정·최근 작업을 자동 표시하되 일반 상위 작업 슬롯의 원격은 사용자가 직접 고정한 작업만 포함하고, 현재 또는 마지막 전환 성공 원격 작업 하나만 별도의 현재/마지막 전환 작업 액션 예외로 허용했습니다. 해당 원격 버튼은 Codex 사이드바 또는 통합 검색을 통해 올바른 컴퓨터까지 활성화합니다.
- 원격 요약의 최근 갱신 시각을 완료 경계로 사용하지 않도록 수정했습니다. 완료 시간은 명시적 수명 주기 종료값 또는 확인된 실시간 `active` → `notLoaded` 전환을 우선하며, 일시적 연결 끊김은 무시하고 새로 시작한 뒤 신뢰할 종료값이 없으면 임의의 짧은 시간을 표시하지 않습니다.
- Codex 작업 행·작성창 포커스·받아쓰기 중지·보내기의 합성 마우스 클릭을 정확한 손쉬운 사용 포커스와 키보드 활성화로 교체해, 해상도와 무관하게 동작하며 작업 버튼이 포인터를 움직이지 않도록 했습니다. 작업 버튼을 길게 누를 때는 임계점을 지난 뒤 기다리지 않도록 작성창 포커스를 임계점 근처에서 비동기로 미리 준비합니다.
- 첫 완료 펄스 프레임을 화면에 보이는 모든 ThreadDeck 소유 버튼에 보장한 뒤 후속 애니메이션 전송량을 낮춰 Neo 갱신 안정성을 높였습니다.
- 현재 키보드 입력 소스와 무관하게 음성 입력을 시작하고, 버튼을 놓으면 현재 Codex의 화면 중지 컨트롤로 녹음을 끝내며, 오디오 입력 시작 실패도 감지하도록 수정했습니다.
- 작업 목록을 일시적으로 읽지 못하면 짧게 재시도하고 마지막 정상 목록을 유지해 오류 카드가 순간적으로 깜빡이지 않도록 했습니다.
- 화면에 보이는 모든 ThreadDeck 페이지에서 주간 한도를 미리 조회하고 마지막 정상 값을 유지해, 한도 페이지를 열 때 발생하던 수 초 대기를 없앴습니다.
- 받아쓰기 종료 후 작성창 포커스를 복원하고 Codex의 화면 전송 컨트롤을 사용하며, 작성 내용이 실제로 비워졌는지 확인한 뒤 필요할 때만 Enter로 안전하게 재시도하도록 보강했습니다.
- 손쉬운 사용 권한 없음, 원격 제목 중복, 원격 작업 접근 불가를 구분하는 버튼 피드백과 완료 전파·음성 이벤트 회귀 검증을 추가했습니다.
- 단일 파일이던 플러그인을 의존성 없는 CommonJS 도메인·파서 모듈로 분리하고, 서로 연관된 화면이 전역 Codex 상태를 따로 읽지 않도록 갱신마다 하나의 스냅샷을 공유합니다.
- 긴 원격 로그에서 중간 단계 이벤트가 누락되지 않도록 UTF-8 안전하고 크기가 제한된 줄 유지와 파일 교체 감지를 적용하고, 타입이 명확한 원격 로그 파서·리듀서, 마지막 정상 사이드챗 스냅샷, 이전 전송이나 다른 작업 버튼의 늦은 결과를 격리하는 전역 음성 제출 세션 토큰으로 비동기 상태 처리를 강화했습니다.
- `node:test` fixture 검증과 소스·배포 번들의 바이트 일치 검사를 추가하고, 검증된 산출물의 정확한 스테이징 복사본을 소스 디렉터리 변경 없이 포장하도록 릴리스 패키징을 단일 빌드로 변경했습니다.

## 0.2.0-beta — 2026-07-18

- 모든 작업 버튼에 길게 눌러 음성 입력 기능을 추가했습니다. 작업 열기, 누르는 동안 녹음, 받아쓰기 대기, 버튼을 놓은 뒤 자동 제출을 한 번에 처리합니다.
- 현재 Codex Desktop 세션 범위에서 임시 사이드챗을 감시하고, 닫힌 사이드챗이 다시 나타나지 않도록 보강했습니다.
- 관찰 구간 겹침, 시작 유예 시간, 대기열 진행 펄스, 모든 ThreadDeck 소유 버튼 적용으로 완료 감지를 강화했습니다.
