## 미배포

## 0.5.0-beta — 2026-07-21

- 사용자의 동작 의도와 Codex에 전달하는 방식을 분리하는 Micro 우선 제어 계층을 도입했습니다. 보호된 로컬 브리지가 연결되면 Effort, Fast mode, 사이드챗, 일반 보내기, 새 작업, 눌러 말하기, Micro 기본 작업 슬롯 6개를 Codex 내부 렌더러 이벤트로 실행하고, 기존 손쉬운 사용·keybridge 경로는 검증된 폴백으로 유지합니다.
- 현재 작업, 작성창의 Effort·Fast 설정, 화면 테마, 기능 목록, Micro 슬롯 6개만 읽는 개인정보 제한형 스냅샷을 추가했습니다. 8개 작업 대시보드와 대기열·목표·시간·완료 판정은 계속 ThreadDeck이 담당하므로 네이티브 Micro 제어와 더 넓은 ThreadDeck 모니터링을 의도적으로 결합합니다.
- 이미 Codex에 전달됐을 가능성이 있는 명령은 결과를 검증하되 기존 방식으로 다시 실행하지 않는 안전 규칙을 추가했습니다. 브리지가 확실히 없거나 해당 기능이 확실히 지원되지 않을 때만 폴백하므로 중복 전송·중복 사이드챗·이중 토글을 막습니다.
- 보호된 Micro 브리지 준비 절차를 추가했습니다. ThreadDeck은 꺼진 Codex를 임의로 실행하지 않고, 이 버전을 처음 설치할 때 이미 실행 중이던 Codex 프로세스도 그대로 둡니다. 사용자가 나중에 Codex를 정상 종료하고 다시 열면 무작위 `127.0.0.1` 디버깅 포트로 한 번만 재실행할 수 있으며 재시도 간격을 제한합니다. `THREADDECK_DISABLE_MICRO_BOOTSTRAP=1`로 이 동작을 끌 수 있습니다.
- 읽기 전용 진단 도구와 구조·설치·문제 해결·보안·이식·오픈소스 문서를 Micro 어댑터 기준으로 보강했습니다. MIT 라이선스 Codex Deck에서 적용·수정한 런타임 연결 방식은 저장소와 배포 플러그인 양쪽에 출처와 전문을 포함합니다.

- 작업 키로 전환할 때 대상 작성창까지 포커스를 옮깁니다. 일반 작업을 열면 사이드챗 포커스 임대를 먼저 해제한 뒤 해당 작업 작성창에 커서를 놓고, 사이드챗을 열면 오른쪽 사이드챗 작성창에 커서를 놓아 입력·보내기·받아쓰기·Fast mode·추론 강도가 방금 누른 키를 따라갑니다.
- 플러그인이 시작되거나 Codex App Server가 다시 시작될 때 Codex의 읽기 전용 데스크톱 설정과 모델 캐시에서 추론 단계 목록을 미리 읽습니다. 이제 선택적인 `Max`·`Ultra`가 있다고 추정하지 않으면서도 첫 클릭부터 버튼이 즉시 반응하고, 실제 적용 시점의 UI 스캔을 최종 기준으로 사용합니다. 실험적 App Server 갱신 호출 자체는 1ms 미만이었지만 현재 Codex Desktop은 외부 클라이언트가 안전하게 연결할 수 없는 전용 stdio 서버를 사용하므로, 별도 서버를 띄우거나 세션 파일을 쓰지 않고 1.1초의 최종 입력 병합 시간을 유지합니다.
- Effort를 바꿀 때마다 현재 Codex 모델·계정에 실제로 보이는 단계를 먼저 스캔하고 그 목록 안에서만 이동하도록 바꿨습니다. 따라서 선택적인 `Max` 단계가 없으면 건너뛰고, 보일 때만 사용하며, 빠른 연타도 같은 실제 목록 위에서 한 번의 왕복 이동으로 합칩니다. 슬라이더형 화면에서는 눈금을 직접 조작하거나 추정하지 않고 Codex의 정확한 `Advanced`를 연 뒤 단계 목록에서 선택합니다. 입력 권한이 정상일 때 선택기 모양만 달라 발생한 실패는 추론 강도 버튼에만 표시하며, 모든 키에 `입력 점검` 경고를 번지게 하지 않습니다.
- 프롬프트 기록뿐 아니라 Codex의 정확한 `thread/fork` → `thread/inject_items` 식별자 쌍으로 열려 있는 모든 사이드챗을 찾습니다. 한 부모 작업 아래 여러 사이드챗이 각각 별도 버튼을 유지하고, `thread/unsubscribe`가 기록되면 실제로 닫힌 사이드챗 하나만 제거합니다.
- 대기 중인 후속 요청을 활성 창 전체로 합산하지 않고 일반 작업·사이드챗 작성 영역의 감싸는 행 위치로 분리합니다. 대기 요청이 실행 단계로 넘어갈 때도 해당 작업 버튼의 숫자만 감소합니다.
- 전용 마이크에 새 사이드챗 작성창을 위한 임시 연결 단계를 추가했습니다. Codex가 새 임시 작업 UUID를 아직 만들지 않았어도 손쉬운 사용 컨트롤 쌍과 창 내부 위치로 오른쪽 사이드챗 작성창을 확인해 그곳에서 받아쓰기를 시작하고, UUID가 나타나는 순간 같은 녹음 세션을 실제 작업으로 인계합니다. 보내기·Fast·추론 강도처럼 작업 식별이 필요한 동작은 기존처럼 부모 작업으로 새지 않고 정확한 식별자를 기다립니다.
- 추론 강도 버튼 연타를 마지막 한 번의 Codex 적용으로 합쳤습니다. 버튼 트랙은 누를 때마다 즉시 움직이지만, ThreadDeck은 입력이 1.1초 멈춘 뒤 마지막으로 정지한 Effort만 직접 선택하므로 중간 단계의 메뉴 조작을 전부 재생하지 않습니다.
- 설정이 비어 있는 새 이전/다음 페이지 키가 반대쪽 페이지로 넘어가던 문제를 고쳤습니다. 이제 정확히 같은 기기에 현재 보이는 액션 구성으로 Neo 페이지를 판별하며, 페이지가 모호할 때는 잘못 전환하지 않고 중단합니다.

## 0.4.0-beta — 2026-07-21

- ThreadDeck 기본 언어를 영어로 바꾸고 Stream Deck의 `application.language`에 따라 영어/한국어를 자동 선택하도록 했습니다. 하나의 플러그인이 액션 목록(`en.json`/`ko.json`)과 렌더러가 그리는 상태·활동·음성·권한·피드백 문구 전체를 현지화합니다.
- 언어가 섞여 있던 수명주기 레코드를 안정적인 활동 코드로 바꾸고, 등록 정보·플랫폼 파싱을 `runtime-info.js`로 분리했으며 향후 Windows 이식에 필요한 어댑터 경계를 문서화했습니다. 이번 릴리스에서 실제 선언·지원하는 런타임은 계속 macOS뿐입니다.
- 저장소의 스크린샷과 GIF 5개를 영어 전용 UI 문구로 다시 만들고 `Thinking`, `Reading logs`, `Checking code`, `Draft ready`처럼 Neo 크기에 맞는 짧은 활동 표현을 적용했습니다. 한국어 문서는 완전한 보조 미러로 유지합니다.
- 한영 설치 안내, 읽기 전용 `pnpm run doctor`, 현지화한 액션 툴팁, manifest Support URL, Apple Silicon/Intel 공용 릴리스 패키지를 추가해 다른 Mac 설치 흐름을 단순화했습니다.
- Codex Stream Deck, Stream Deck Neo, Codex Micro, 음성 받아쓰기, AI 에이전트 워크플로 검색 문구와 관련 프로젝트 출처를 보강했습니다. 포함된 MIT Codex Micro 프로토콜 스냅샷은 연구 전용이며 런타임 번들에 들어가지 않는지 검증합니다.

- Stream Deck이 처음 연결될 때 현재 작업 카드만 먼저 채워지고 Effort/Fast·마이크·보내기 등이 아직 그 작업에 연결되지 않던 초기화 경합을 고쳤습니다. 이제 완전히 수명주기 정보가 채워진 현재 작업 스냅샷을 모든 작업 종속 버튼에 한 번에 연결하고 Effort 표시도 즉시 다시 그립니다.
- 추론 강도/Fast 결합 버튼을 0.6초 이상 누르면 버튼을 놓을 때까지 기다리지 않고 임계점에서 Fast mode 전환을 즉시 시작합니다. 이후 버튼을 놓는 동작은 이미 시작된 전환을 다시 실행하지 않습니다.
- 실행 중인 턴 메타데이터와 현재 작성창 설정을 분리했습니다. 작업 카드 상단은 실행 시작 때 확정된 Effort와 Fast/일반 값을 끝까지 유지하고, 결합 버튼은 Codex에서 직접 바꾼 값까지 따라가며 다음 응답의 설정을 나타냅니다. 대기 후속 요청은 Codex가 실제로 실행을 시작할 때의 작성창 설정을 사용하고, 그때 확정된 새 턴 값이 작업 카드로 넘어갑니다.
- `Max` 위의 `Ultra`를 실제 선택 가능한 끝 단계로 고쳤습니다. 해당 Effort 선택 직후 Codex의 `Use Ultra with Full access?` 경고가 뜨면 고정 제목·설명·두 버튼을 모두 정확히 대조한 뒤 `Use Full access`만 활성화합니다. 일반 확인창을 임의 승인하거나 작업 권한을 제한 모드로 바꾸는 `Continue` 경로는 누르지 않습니다.
- 추론 강도 버튼을 빠르게 여러 번 눌러도 각 입력을 즉시 화면에 반영하고 하나도 버리지 않도록 바꿨습니다. Codex 메뉴 조작은 뒤에서 안전하게 직렬 적용되며, 끝 단계에서는 기존처럼 방향을 반대로 전환합니다. 적용 중 작업이 바뀌면 다른 작성창을 건드리지 않고 실패 폐쇄합니다.
- 작업 완료 펄스의 키별 갱신률을 약 5–6fps에서 전체 알림 25fps, 미확인 작업 호흡 30fps로 높였습니다. 모든 키가 같은 위상으로 움직이며, 완료 작업의 강한 3회 호흡과 전체 버튼의 2회 알림 모두 부드러운 가속·감속 곡선으로 시작하고 끝납니다.
- 추론 강도/Fast 결합 버튼을 큰 현재 단계명과 작은 6단계 트랙 중심으로 정리하고 모호한 방향 화살표를 없앴습니다. 작은 슬라이더를 익명의 1픽셀 손쉬운 사용 그룹으로 노출하는 Codex 버전도 인식하되, 확인되지 않은 다음 컨트롤로 Tab 이동하지 않고 실패 폐쇄합니다. Effort 버튼이 옆의 마이크를 잘못 켜는 문제도 함께 막았습니다.
- 작은 슬라이더의 확인된 `Advanced`를 우선 열어 현재 모델이 실제 제공하는 모든 추론 단계를 버튼 목록에서 선택합니다. 선택 뒤에는 정확한 목표 단계가 읽힐 때까지 기다려 `Max`처럼 늦게 갱신되는 끝 단계가 거짓 실패로 표시되지 않습니다. 일반 속도에서는 번개를 숨기고 Fast일 때만 트랙과 같은 색의 번개 및 계속 움직이는 파티클을 결합 버튼과 작업 카드 상단에 표시하며, Ultra가 아닌 단계에서도 애니메이션을 갱신합니다. 기본 대시보드도 한도·새 작업·사이드챗·보내기 / 현재 작업·추론+Fast·마이크·뒤로가기 배열로 바꿨습니다.

- 기본 대시보드의 새 작업 자리를 추론 강도 버튼으로 교체하되 새 작업 액션은 사용자 지정 프로필용으로 유지했습니다. 짧게 누르면 트랙을 다음 단계로 즉시 움직인 뒤 현재 모델의 실제 `Effort` 하위 메뉴에서 가능한 단계 하나를 적용하고, 검증 결과와 다르면 보정하거나 실패 시 원래 값으로 복구합니다. 같은 버튼의 작은 번개는 확인된 속도를 표시하고 0.6초 길게 누르면 Fast mode를 전환하며, Fast 전용 액션은 일반 클릭으로 바로 전환합니다.
- Codex 앱에서 직접 작업을 바꾸면 대시보드 현재 작업을 1초 안에 갱신하는 가벼운 활성 창 감시를 추가했습니다. 보내기·전용 마이크·Fast mode·사이드챗도 누르는 순간 같은 현재 작업을 다시 확인하고, 보내기와 마이크는 해당 작성창에 초점을 맞추며 새 사이드챗은 그 작업을 부모로 기록합니다. 새 사이드챗 ID와 실제 포커스가 확인될 때까지 전환 임대를 유지하므로, 바로 이어 누른 마이크·보내기·Fast mode·추론 강도가 이전 작업에 적용되지 않습니다. 전용 마이크의 상태 표시가 오래된 마지막 전환 작업을 가리키던 문제도 고쳤습니다.
- 권한·기능 이상을 먼저 알리는 상태 감시를 추가했습니다. ThreadDeck이 시작할 때와 30초마다 macOS 손쉬운 사용, 합성 키 입력 권한, 읽기 전용 Codex AX 접근을 따로 검사하고, 권한이 없으면 반복 방지 간격을 둔 macOS 공식 권한 요청과 버튼 경고를 표시하며, 복구도 자동으로 확인합니다. 이제 합성 키가 버려져도 네이티브 명령이 성공한 것처럼 끝나지 않고 권한별 종료 코드를 반환합니다. 권한은 정상인데 Codex 조작이나 키 입력이 두 번 연속 실패하면 짧은 오류로 사라지지 않고 계속 보이는 점검 경고로 전환합니다.
- 최신 macOS에서 전역 지금 재생 중 API가 실제 재생 중인 오디오를 `paused`로 잘못 보고해 음성 입력 중 미디어가 멈추지 않던 문제를 고쳤습니다. 이제 앱 허용 목록 없이 활성 Core Audio 프로세스의 화면 앱을 찾고, 의미가 확인된 일시정지·재생 컨트롤을 사용하며, 짧게 유지되는 로컬 임대 정보에 기록한 앱만 재개합니다. Apple Music과 Chrome·Safari YouTube를 실기 검증했습니다. 미디어 제어 상태를 끝내 확인하지 못하면 조용히 무시하지 않고 `미디어 점검` 경고를 유지합니다.
- Fast mode 전환이 끝날 때 남아 있는 속도 메뉴를 먼저 닫고 Chromium이 교체한 정확한 Codex 작성창에 포커스를 다시 맞춥니다. 이제 번개 버튼 직후에도 보내기·받아쓰기·후속 단축키가 바로 작동합니다.

- `node:test` fixture 검증과 소스·배포 번들의 바이트 일치 검사를 추가하고, 검증된 산출물의 정확한 스테이징 복사본을 소스 디렉터리 변경 없이 포장하도록 릴리스 패키징을 단일 빌드로 변경했습니다.
